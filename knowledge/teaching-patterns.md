# MC Wizard teaching patterns

When a child knows the desired behavior but not its name, restate the behavior first and then name two or three possible mechanisms. For example, “a button that alternates between on and off” suggests a T flip-flop; “a short pulse when a signal changes” suggests an observer or monostable circuit.

Teach with the smallest visible experiment:

1. Predict what will happen.
2. Build or place only the minimum parts.
3. Try one input.
4. Observe the output.
5. Change one variable and predict again.

Explain a new term with an everyday analogy, but connect the analogy back to the exact blocks. A T flip-flop is like a clicky lamp switch: a brief press changes the remembered state.

Offer a compact demo before a large build. Protect existing work, state where the demo will be placed, and provide an undo path before any destructive or large action.
